PROGRAMACION MULTIHILOS EN JAVA.

La programación multihilo en Java se refiere al uso de múltiples hilos de ejecución para lograr la concurrencia dentro de un programa. En Java, los hilos permiten que partes diferentes del código se ejecuten de forma independiente, lo que puede mejorar el rendimiento y la capacidad de respuesta de una aplicación al realizar múltiples tareas simultáneamente.

Creación de hilos: Utilizando la clase Thread o implementando la interfaz Runnable. Puedo iniciar un hilo llamando al método start() del objeto Thread.

Sincronización: En situaciones donde múltiples hilos acceden a los mismos recursos compartidos, la sincronización es crucial para evitar condiciones de carrera y garantizar la consistencia de los datos. La palabra clave synchronized, los bloqueos lock y las clases del paquete java.util.concurrent son herramientas para lograr la sincronización.

Métodos y conceptos clave: join(): Permite que un hilo espere a que otro hilo termine su ejecución. wait() y notify()/notifyAll(): Métodos utilizados para la comunicación entre hilos mediante la espera y la notificación. volatile: Palabra clave utilizada para garantizar que los cambios en una variable sean visibles para otros hilos.

Gestión de hilos: ThreadPoolExecutor: Permite la gestión de un grupo de hilos, reutilización y control del número máximo de hilos en ejecución simultánea. ExecutorService: Interfaz para administrar la ejecución de tareas asincrónicas.

CODIGO EN JAVA

creamos una clase Empleado que representa a un trabajador (un hilo) en una heladería. Cada empleado realiza 5 tareas simuladas, imprimiendo mensajes que indican en qué tarea están trabajando. Se simula una tarea que toma tiempo con Thread.sleep(1000), donde cada iteración del bucle se imprime un mensaje y se espera un segundo antes de pasar a la siguiente tarea. Al final, se imprime un mensaje cuando cada empleado (hilo) completa todas sus tareas. En la clase Heladeria, creamos dos empleados (dos hilos con nombres diferentes, Juan y María) y los iniciamos (start()) para que comiencen a trabajar simultáneamente en sus tareas.
